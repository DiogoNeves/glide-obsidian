# Concept

Glide turns an Obsidian vault into agent-readable context.

It is not a hosted app, a replacement for your notes, or a specific model setup. Glide is a set of defaults: plain Markdown instructions, memory files, checklists, skills, and automation prompts that tell an agent how to help without losing the shape of your thinking.

You choose the execution layer. That might be Codex, Claude Code, another cloud model, a local model, connectors, automations, or a mostly manual workflow.

## The Idea

Most people already have useful context scattered across notes, emails, calendars, tasks, documents, screenshots, and half-formed ideas. Agents can help, but they need a durable place to learn what matters and how to act.

Glide gives that context a home:

- `Agent HQ/` for the agent's memory and operating structure
- areas for the parts of life or work that need ongoing attention
- skills for repeatable ways of thinking
- checklists for reviews, decisions, goals, research, and drift
- automations for lightweight recurring attention

The goal is better defaults. Focus on what you care about. Let the agent watch, research, summarize, remind, and maintain more of the rest.

## Who It Is For

Glide is for people who want to manage their own context better:

- creators turning rough notes into useful work
- solopreneurs juggling ideas, customers, finances, and operations
- researchers and writers building from long-term notes
- operators who want recurring reviews without building a SaaS stack
- anyone who wants agents to help without surrendering the system to an opaque app

## What Makes It Self-Evolving

Glide can change because its behavior is written down.

When something is annoying, useful, too noisy, too passive, or too risky, the user can ask the agent to update the relevant instruction, checklist, skill, or automation prompt. Drift review keeps those changes aligned with the core principles.

The system should evolve like software, but remain readable like notes.

## Boundaries

Glide memory and agent workings stay in `Agent HQ/`. Ordinary notes remain ordinary notes unless the user explicitly asks to edit them.

Glide itself does not decide where computation happens. The harness, model, connectors, sync, and automations chosen by the user determine data exposure and execution behavior.
